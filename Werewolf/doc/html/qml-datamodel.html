Detailed Description

The DataModel is a Singleton object, that provides functions for adding, removing, modifying and finding players, as well as the ListModel of the players that is used by the AppListView in WerewolfMainPage.

On startup it loads the players from the localStorage and populates the ListModel.

Property Documentation

playersListModel : alias

Contains all players that are in the game.

The playersListModel is, as the name suggests, a ListModel of the players that is used in the AppListView in the AppPlayerPage. This ListModel is directly referenced by the AppListView's model property.


roles : alias

Contains all possible roles.

It contains a list of all possible roles including their properties and a set of functions that are all connected to roles.


Signal Documentation

availabilityUpdated(var availabilityInformation)

Sends out information about roles' availability.

In Werewolf certain roles should only be assigned to a single player per game (Like the Witch or the Seer). This signal takes care of that by letting the roleChooser in PlayerPage know which roles are available.

Every time the player data is modified (when a player is added, removed or modified) it notifies its subscribers about the updated availabilities of the roles. It does that by sending an object like this:

{
    "roleName": isAvailable("roleName"),
    ...
}

Method Documentation

addPlayer(jsobject player)

Assigns player an ID and adds it to the playersListModel and the localStorage.

Before it does that it checks whether the player is valid or not. If it's not a valid player object it simply stops (returns).


getHighestId()

Returns the highest assigned id or -Infinity if there are no players in the game.


getNextPlayerId()

Returns the next unassigned playerId.


getPlayerById(int playerId)

Returns the player object with the given playerId or null if the playerId is not assigned to any player.


getPlayersByRole(string role)

Returns a list of players with the given role.


isValidPlayerModel(jsobject player)

Returns true if the name attribute isn't empty and the role attribute is a possible role.


modifyPlayer(jsobject player)

Updates player in the playersListModel and the localStorage.


removePlayer(int playerId)

Removes player with the provided playerId from the playersListModel and the localStorage.
